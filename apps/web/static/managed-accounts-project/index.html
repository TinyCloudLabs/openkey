Now designing
The important status nuance

The control plane is secure and callable today. /dashboard/organizations already supports organization creation and entitlement viewing, but a user cannot yet visit /console and operate the full tenant surface.

Ecosystem model

Personal custody and tenant operations stay separate.

OpenKey hosts identity, passkeys, consent, and eject. Tenant apps initiate managed registration. Operators manage their organization through a separate session-authenticated console.

Signup and custody

The app starts the journey.
OpenKey earns the trust.

There is no separate tenant-owned signup UI. The tenant app creates a registration intent, then sends the user to an OpenKey-hosted passkey and consent flow.

App requests registration

The tenant backend uses its provisioner credential to create a short-lived intent with the app identity and return URL.

OpenKey verifies the user

The user sees the app name and custody model, then creates or uses an OpenKey passkey on an OpenKey origin.

Managed account becomes ready

The app receives a safe completion result. Custody begins under the tenant broker and the account appears in the console.

User manages connected apps

The personal dashboard currently calls this “Managed accounts.” The console release should rename it “Connected apps,” while preserving custody state and the “Leave app” action.

Eject switches custody

OpenKey validates the user, transitions to personal custody, invalidates tenant authority, and emits lifecycle events.

Entitlement model

One product model,
three operational tiers.

The tier structure is implemented. Exact prices and several commercial quotas remain business decisions; the platform should not invent them in UI copy.

Free

Build and validate

  • 1 app · 3 members
  • 100 managed accounts / MAU
  • 60 requests per minute
  • 7-day audit retention
Pro

Operate in production

  • 10 apps · 25 members
  • 10,000 managed accounts / MAU
  • 600 requests per minute
  • 90-day audit retention
Enterprise

Govern at scale

  • 1,000 apps · 10,000 members
  • 10M managed accounts / MAU
  • 10,000 requests per minute
  • 2,555-day audit retention
Limits are code; pricing is not.

These are the configured v1 entitlement defaults. App, account, policy, and webhook limits are enforced today; member count, MAU, request rate, and audit retention are stored and returned but still need runtime enforcement. Exact prices, public quota labels, Enterprise terms, and the legacy SCALE migration path need product approval.
